There are a few reasons I still work without AI. There are clients who do not allow AI in their codebase. For this client, things are as they have always been. Work on the feature and code everything. Having no AI is actually hard to do right now. Not because I depend on it to think for me. But before Google's auto AI summaries in Google search results, you would Google a problem you are stuck on, find the blog post or Stack Overflow post that closely matched your problem. Then you could try to apply that partial fix to your situation. Now with Google automatically presenting you with the AI answer to your search, there really isn't any need to go to Stack Overflow anymore. Google surfaces the best — sometimes — answer right as the top result. Most of the time this answer came from Stack Overflow.

In this case I'm still handwriting the code. All architecture decisions fall on me and the team of humans. I'd want to talk through a problem and do it with the team members. Not much has changed, but this style of development is objectively slower in the beginning than agentic assisted programming. That's due to the time I have to think through a problem or solution I don't know. This takes time, but the trade-off is worth it to me. I understand 100% of the code written and everything the program is doing.


The trade-off is worth it. I want to understand the application inside and out while still using my brain. If I am doing all the research, planning, coding and testing that means I'm in the project 100%. There are no surprises, unknowns or assumptions about what's going in the project. This keeps me thinking and makes me feel safe that I won't introduce a vulnerability I can't catch. A worry that I have, as well as other developers, is that it's so easy to outsource everything to AI. When I let AI do all the hard work I <a href="https://softwaredoug.com/blog/2025/03/28/ai-brainrot-opportunity" target="_blank">risk suffering AI brain rot</a>. Developing everything keeps me sharp and on top of the application. 

However, there are drawbacks to not using AI. One is being slower. Application development speeds up by using an AI agent. Not blindly, of course, but Claude can type faster and create a bootstrapped .Net Core WebAPI with OData integration based on a database model in less time than I can create the Home controller. I still need to check the code for accuracy but to get up off the ground or to write a function, AI is an incredible paired developer.
